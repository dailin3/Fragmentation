# 信息提取规则

本文件用于指导 AI 从日记/笔记中提取值得保存的碎片知识点，并归入 Knowledge Tree 的对应子领域。

## 核心原则
- **只提取有长期价值或可复用的内容**，不保留流水账和瞬时感想。
- **每个提取的知识点应能独立成块**，包含足够的上下文，方便未来检索和理解。

## 具体提取规则

### 1. 技术学习相关的概念和方法
- 新学的编程语言特性、库、框架、API 用法。
- 算法、数据结构、设计模式、架构原则的实际应用。
- 调试过程中的关键发现（如某个 bug 的根本原因）。
- 开发工具、命令行的技巧（如 Git、Docker、Obsidian 插件）。
- 计算机基础知识（操作系统、网络、数据库、数学）的实例化理解。

### 2. 个人成长中的反思和洞察
- 对自身学习效率、时间管理、自律情况的复盘，且得出可操作的改进点。
- 关于完美主义、拖延、情绪波动的深层认识及应对策略。
- 决策过程的回顾（为什么选择 A 而不是 B，结果如何）。
- 人际关系、家庭、朋友互动中获得的启发（如沟通技巧、边界感）。
- 价值观、信念、人生目标的澄清或转变。

### 3. 具体的项目经验和问题解决过程
- 完成一个项目（如 C++ 项目、爬虫、开源贡献）的关键步骤、难点与解法。
- 实践中的教训（如“过早优化导致返工”、“测试覆盖不足”）。
- 可复用的代码片段、配置模板、自动化脚本。
- 需求分析、架构设计、重构的实例记录。

### 4. 情绪管理和自我认知
- 记录情绪触发点、身体反应、后续调节方法（如冥想、运动）。
- 梦境分析，如果与现实压力或潜在想法相关。
- 关于勇气、梦想、童年经历的反思，且与当前成长关联。
- 注意力管理、心流体验的具体案例。

### 5. 经济、金融与商业的实用认知
- 投资/理财决策的逻辑（如买某只股票的原因、复盘）。
- 对市场、商业模式的观察分析（例如某个 App 的盈利方式）。
- 对资本、资产、剩余价值的个人理解，结合生活实例。

### 6. 哲学、道德与规则的思辨
- 对决定论、自由意志、理性、智慧的个人见解，且有一定论证。
- 道德困境或规则冲突的真实经历与处理方式。
- 对“机会”、“责任”、“创新”的深度思考。

### 7. 学习策略与知识管理方法
- 有效的阅读方法（如主题阅读、卡片笔记）。
- 使用 Obsidian、信息流管理、RSS 订阅的实践心得。
- 自学路径规划、课程评价、书籍推荐（需给出具体理由）。

## 不应提取的内容（负向规则）

- 日常琐事：起床时间、吃了什么、通勤路线、无关紧要的购物记录。
- 一句话感想：如“今天好累”、“Python 真难” 没有展开。
- 纯粹的情绪发泄，没有分析和转化。
- 重复出现且无新信息的记录（如连续几天同样抱怨某件事）。
- 完全复制的第三方内容（如复制粘贴的新闻、教程），除非有自己的批注或总结。

## 澄清提问规则

- 仅当日记内容包含**有价值信息但不完整**时才提问，例如某个概念只提到了名字但缺少背景。
- **不要要求用户提供日记原文**（输入本身就是日记全文，无法补充）。
- 如果日记本身内容太少且没有具体知识线索，直接跳过，不提问、不提取。
- 提问要有针对性，具体到某个知识点、决策或经验。
